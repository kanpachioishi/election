## 2. 重要な固定判断

### 2.1 プロダクトの定義

- このサイトは政治ニュースサイトではない
- 常設型の生活者向け選挙ナビとして作る
- 核は「自分の地域の次の選挙に最短で辿り着けること」

### 2.2 MVP の核

- 入口は郵便番号
- 住所自由入力は後回し
- 画面は 5 本に絞る
  - トップ
  - 地域ページ
  - 個別選挙ページ
  - 全国スケジュール一覧
  - 投票率データ
- 候補者、公報、投票所、期日前投票は独立画面にしない
- 確認済みリンクだけを個別選挙ページに出す

### 2.3 安全性の核

- `source_url`, `confirmed_at`, `status` は公開データの必須項目
- `status != verified` の情報を主要表示に使わない
- 郵便番号が曖昧なときに自治体を決め打ちしない
- 公開前監査を通らない限りリリースしない
- 古い情報を最新のように見せない

### 2.4 カバー方針

全国保証するもの:

- 郵便番号データ
- 地域マスタ
- upcoming な国政選挙の基本情報
- upcoming な都道府県選挙の基本情報
- 投票率ページに使う国政基礎統計

確認済みから段階展開するもの:

- 市区町村選挙
- 補欠選挙
- 候補者一覧リンク
- 選挙公報リンク
- 期日前投票リンク
- 投票所リンク

## 3. データ方針の現時点の結論

正本データは `data/v1/` 配下の JSON に置く方針で固定した。

想定ファイル構成:

```text
data/
  v1/
    regions.json
    elections.json
    postal_code_mappings/
      {prefix3}.json
    election_resource_links/
      {election_id}.json
```

大きなルール:

- キー名は `snake_case`
- コード値は文字列保持
- `verification` は全公開データに必須
- `id` は機械参照用
- `slug` は URL 用
- 旧 [regions.js](C:/Users/shimo/Desktop/election/data/regions.js) と [elections.js](C:/Users/shimo/Desktop/election/data/elections.js) は正本ではなく互換用派生ファイル扱い
