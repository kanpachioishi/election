# わたしの選挙 画面一覧

最終更新: 2026-04-08
ステータス: Draft v1
前提: [PRD](C:/Users/shimo/Desktop/election/docs/PRD.md)

## 1. この文書の目的

この文書は、PRD をもとに「MVPで本当に必要な画面」を定義するための画面一覧である。

この文書で決めること:

- どの画面を MVP に含めるか
- 各画面の役割は何か
- どの情報をその画面で見せるか
- 主な遷移先はどこか

この文書でまだ決めないこと:

- 詳細 UI デザイン
- コンポーネント単位の見た目
- データスキーマの厳密な定義

## 2. 画面設計の原則

1. 画面数より導線の短さを優先する
2. 専用ページを増やす前に、個別選挙ページ内で完結できるかを考える
3. 候補者、公報、投票所、期日前投票は、MVPでは独立画面に分けすぎない
4. 郵便番号が曖昧な場合やデータ未整備の場合も、行き止まりにしない
5. 終了済み選挙は、現役情報と混同しない

## 3. MVP 画面の結論

MVP の主画面は 5 つに絞る。

1. トップページ
2. 地域ページ
3. 個別選挙ページ
4. 全国スケジュール一覧ページ
5. 投票率データページ

加えて、独立画面ではなく重要な状態として次の 2 つを定義する。

1. 郵便番号未解決 / 候補地域選択状態
2. 終了済み / アーカイブ状態

## 4. MVP 画面詳細

### 4.1 トップページ

- 画面ID: `S01`
- 役割:
  郵便番号から最短で「自分の次の選挙」に入る入口。
- 主なユーザー意図:
  何のサイトか理解したい。自分の地域の次の選挙をすぐ知りたい。
- 主要情報:
  - 郵便番号検索フォーム
  - このサイトで分かること
  - 直近の注目選挙
  - 信頼性の説明
  - 未掲載/未確認の意味
- 主なCTA:
  - 郵便番号で調べる
  - 全国スケジュールを見る
  - 投票率データを見る
- 主な遷移先:
  - 地域ページ
  - 個別選挙ページ
  - 全国スケジュール一覧ページ
  - 投票率データページ
- MVPで重要な理由:
  入口が弱いと、このサービスの価値である「最短到達」が成立しないため。

### 4.2 地域ページ

- 画面ID: `S02`
- 役割:
  その地域に関係する選挙を時系列でまとめる。
- 主なユーザー意図:
  自分の地域では何の選挙があるかを一覧で見たい。
- 主要情報:
  - 地域名
  - 関係する次の選挙一覧
  - 選挙名
  - 選挙種別
  - 投票日
  - ステータス
  - 出典
  - 最終確認日時
- 主なCTA:
  - 選挙詳細を見る
  - 全国スケジュールを見る
- 主な遷移先:
  - 個別選挙ページ
  - 全国スケジュール一覧ページ
  - トップページ
- MVPで重要な理由:
  郵便番号検索後の着地点であり、「自分ごと化」の中核だから。

### 4.3 個別選挙ページ

- 画面ID: `S03`
- 役割:
  1つの選挙について、投票に必要な最小限の情報をまとめる。
- 主なユーザー意図:
  この選挙について、結局どこを見ればよいか知りたい。
- 主要情報:
  - 選挙名
  - 選挙種別
  - 対象地域
  - 投票日
  - 告示日 / 公示日
  - 基本説明
  - 出典
  - 最終確認日時
  - 公式ページへのリンク
  - 確認済みの候補者 / 公報 / 投票所 / 期日前投票リンクまたはセクション
  - 未掲載 / 未確認の明示
- 主なCTA:
  - 公式情報を見る
  - 地域ページに戻る
- 主な遷移先:
  - 地域ページ
  - トップページ
  - 外部の公式ページ
- MVPで重要な理由:
  MVPにおける最重要画面。専用ページを増やす前に、まずここで必要情報を整理する。

### 4.4 全国スケジュール一覧ページ

- 画面ID: `S04`
- 役割:
  全国の選挙を横断で確認できる一覧。
- 主なユーザー意図:
  全国でどんな選挙が予定されているか見たい。地域をまたいで探したい。
- 主要情報:
  - 選挙一覧
  - 選挙名
  - 地域
  - 選挙種別
  - 投票日
  - 残り日数
  - ステータス
  - 絞り込み
- 主なCTA:
  - 選挙詳細を見る
  - 地域ページを見る
- 主な遷移先:
  - 個別選挙ページ
  - 地域ページ
- MVPで重要な理由:
  SEOの入口にもなり、地域検索以外の探索導線も担えるため。

### 4.5 投票率データページ

- 画面ID: `S05`
- 役割:
  投票率データを通じて、関心喚起と理解補助を行う。
- 主なユーザー意図:
  投票率の推移や年代差を知りたい。
- 主要情報:
  - 過去の投票率推移
  - 年代別投票率
  - 簡単な解説
  - 出典
- 主なCTA:
  - トップへ戻る
  - 全国スケジュールを見る
- 主な遷移先:
  - トップページ
  - 全国スケジュール一覧ページ
- MVPで重要な理由:
  既存草稿に資産があり、社会的意義や理解促進の文脈を補強できるため。

## 5. MVP の重要状態

### 5.1 郵便番号未解決 / 候補地域選択状態

- 状態ID: `ST01`
- 役割:
  郵便番号が一意に地域へ解決できない場合の受け皿。
- 必要な理由:
  郵便番号が曖昧な場合に自治体を推測で決め打ちしない、という PRD を守るため。
- 表示内容:
  - 解決できなかった理由
  - 候補地域の一覧、または未解決案内
  - 再入力導線
  - 全国スケジュールへの退避導線
- 主な遷移先:
  - 地域ページ
  - トップページ
  - 全国スケジュール一覧ページ

### 5.2 終了済み / アーカイブ状態

- 状態ID: `ST02`
- 役割:
  終了済みの選挙を現役情報と混同させない。
- 必要な理由:
  古いデータを最新のように見せない、という PRD を守るため。
- 表示内容:
  - 終了済みであること
  - 投票日
  - 必要なら結果やアーカイブ導線
  - 現在有効な関連ページへの導線
- 適用先:
  - 全国スケジュール一覧ページ
  - 地域ページ
  - 個別選挙ページ

## 6. MVP外に回す画面

以下は価値はあるが、MVPでは独立画面にしない。

### 6.1 候補者一覧ページ

- 理由:
  全国標準化が重く、まずは個別選挙ページ内の確認済みリンクで十分だから。

### 6.2 候補者詳細ページ

- 理由:
  データ運用コストが高く、MVPの主価値ではないから。

### 6.3 候補者比較ページ

- 理由:
  差別化として魅力はあるが、MVPではスコープが大きすぎるから。

### 6.4 選挙公報専用ページ

- 理由:
  まずは個別選挙ページ内に確認済みリンクを置けばよいから。

### 6.5 期日前投票専用ページ

- 理由:
  MVPでは専用ページ化より、個別選挙ページからの外部導線が優先だから。

### 6.6 投票所案内専用ページ

- 理由:
  投票所データの取得・更新負荷が高く、段階展開に向いているから。

### 6.7 FAQ / 用語集の独立ページ

- 理由:
  内容は必要だが、MVPではトップページや個別選挙ページ内の軽量セクションで足りるから。

## 7. ユーザー導線

MVP の基本導線は次の通り。

1. トップページで郵便番号を入力する
2. 地域ページに到達する
3. 次の選挙を選ぶ
4. 個別選挙ページで投票日と公式情報を確認する
5. 必要に応じて公式ページへ遷移する

補助導線は次の通り。

1. トップページから全国スケジュール一覧へ行く
2. 一覧から個別選挙ページへ行く
3. トップページまたは一覧から投票率データを見る

例外導線は次の通り。

1. 郵便番号が曖昧なら、未解決 / 候補地域選択状態に入る
2. 選挙が終了済みなら、終了済み / アーカイブ状態で見せる

## 8. 画面優先順位

### P0

- トップページ
- 地域ページ
- 個別選挙ページ
- 全国スケジュール一覧ページ
- 郵便番号未解決 / 候補地域選択状態
- 終了済み / アーカイブ状態

### P1

- 投票率データページ
- トップページ内のFAQ / 使い方セクション
- 個別選挙ページ内の確認済み詳細セクション

### P2

- 候補者一覧ページ
- 候補者詳細ページ
- 候補者比較ページ
- 選挙公報専用ページ
- 期日前投票専用ページ
- 投票所案内専用ページ
- 独立したFAQ / 用語集ページ

## 9. 画面一覧の結論

MVP では、主画面を増やしすぎないことが重要である。

このプロジェクトの価値は、候補者ページの数ではなく、

- 自分の地域の次の選挙が見つかること
- その選挙の基本情報と公式導線が見つかること
- データが曖昧なときに誤案内しないこと

にある。

そのため、MVP の画面一覧は「5つの主画面 + 2つの重要状態」を基本構成とする。
