## 実装レビュー観点

- `ElectionResourceLink` はリンクカード用として適切。ページ内チェック情報をすべて `summary` に詰め込むと表示・検証・更新の境界が崩れる。
- 最小追加モデルは election 単位の `official_facts` などの小さな配列で足りる。項目名、値、単位、source_url、last_checked_at、note、derived boolean を持てばよい。
- 候補者一覧・公報は「未掲載」を公式リンクとして登録しない。既存の再確認フローと同じく、公開確認後に `candidate_list` / `bulletin` を追加する。
- 期日前投票期間のような導出値は、原文が制度説明であるため `derived: true` とし、告示日・投票日から導出した旨を表示または検証メモに残す。
- 投票所65件を初期MVPで正規化しすぎない。まずは公式投票所一覧ページへの導線と、注意事項・件数・地図リンクの表示で十分。
