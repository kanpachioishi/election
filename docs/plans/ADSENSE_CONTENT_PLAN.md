# AdSense Content Plan

更新日: 2026-04-19

## 前提

Google AdSense の公式ヘルプでは、審査の主な観点として次を挙げている。

- 十分な独自コンテンツがあること
- 良いユーザー体験とナビゲーションがあること
- 各ページに、訪問者にとって読む価値があること
- クローラがページ内容を理解しやすいこと

参考:

- https://support.google.com/adsense/answer/12176698?hl=en
- https://support.google.com/adsense/answer/7299563?hl=en
- https://support.google.com/adsense/answer/9724?hl=en
- https://support.google.com/adsense/answer/81554?hl=en

このサイトの現状の弱点は、トップ・一覧・通知登録・データ状況が「導線中心」で、ページ自体の読み物価値が薄く見えやすいこと。
逆に、専用の個別選挙ページ 3 本は比較的厚い。

## 最優先でやること

1. トップに独自説明文を 2-3 ブロック追加する
2. 一覧の詳細パネルに、各選挙の短い独自要約を入れる
3. 個別選挙ページの型を主要選挙へ広げる
4. 通知登録ページを「設定画面」から「案内ページ」へ少し寄せる
5. 固定ページとして `このサイトについて / 情報の確認基準 / プライバシー / お問い合わせ` を整える

## ページ別の追加内容

### 1. トップ `site/index.html`

#### 追加ブロックA: このサイトでできること

配置:

- ヒーロー直下、検索セクションの前

文案:

このサイトは、選挙ごとに候補者一覧、選挙公報、期日前投票、投票所案内などの公式情報を見つけやすく整理するための案内ページです。
まずは地域で絞り込み、気になる選挙を選ぶと、確認済みの公式リンクをまとめてたどれます。

役割:

- トップを単なる検索 UI ではなく、独自価値を持つ入口にする

#### 追加ブロックB: 情報の確認方針

配置:

- ブロックAの直後

文案:

掲載している情報は、自治体や選挙管理委員会などの公式発表をもとに順次確認しています。
掲載前後で内容が更新されることがあるため、このサイトでは案内と要点整理を行い、最終確認は各公式ページでできるようにしています。

役割:

- 独自編集と一次情報依拠の両方を明示する

#### 追加ブロックC: 最初に見るべき項目

配置:

- ブロックBの直後

文案:

選挙で最初に確認したいのは、投票日、候補者情報、期日前投票の場所と期間です。
公報や候補者一覧は比較の入口になり、投票所案内は直前の確認に役立ちます。
このサイトでは、まず必要になりやすい順に公式情報へ進めるように並べています。

役割:

- サイトの編集方針をユーザー価値として説明する

### 2. 一覧の詳細パネル `site/assets/app.js`

各選挙ごとに、詳細パネルへ次の 4 要素を追加する。

#### 追加要素A: この選挙でまず確認すること

文案テンプレ:

この選挙で最初に確認したいのは、投票日、候補者情報、期日前投票の場所と期間です。
公報や候補者一覧は比較の入口になり、投票所案内は直前の確認に役立ちます。

#### 追加要素B: いま掲載している公式情報の範囲

文案テンプレ:

このページでは、現在確認できている候補者一覧、選挙公報、期日前投票、投票所案内などの公式情報をまとめています。
公開前の項目や、自治体側でまだ案内が出ていない項目は、確認でき次第追加します。

#### 追加要素C: 確認日と更新メモ

文案テンプレ:

この選挙の基本情報とリンクは、確認日を記録しながら見直しています。
告示前、告示後、投票日前では公開される情報が変わるため、見つからない項目がある場合もあります。

#### 追加要素D: このページの役割

文案テンプレ:

このページは、今回の選挙で必要な公式情報を探し回らなくても済むよう、利用者の行動順に沿って整理した案内ページです。

役割:

- 単なるリンク一覧から、独自編集された選挙案内ページへ寄せる

### 3. 個別選挙ページ `site/elections/*.html` と各 `site/assets/*-2026.js`

既存 3 ページの型を、今後の主要選挙へ横展開する。

各ページに最低限必要な独自本文:

- この選挙ページで分かること
- まず確認したいポイント
- 確認日と更新メモ

#### 追加セクション: この選挙ページで分かること

文案:

このページでは、今回の選挙で確認したい公式情報を一か所にまとめています。
候補者一覧、選挙公報、投票日、期日前投票、投票所案内などを、探し回らなくても確認できるよう整理しています。

#### 追加セクション: まず確認したいポイント

文案:

この選挙で最初に確認したいのは、投票日、候補者情報、期日前投票の場所と期間です。
比較したいときは公報と候補者一覧、投票直前には投票所案内を見るのが分かりやすい流れです。

#### 追加セクション: 確認日と更新メモ

文案:

このページのリンクと基本情報は、最終確認日を記録しながら見直しています。
候補者確定前、告示後、投票日前では公開される情報が変わるため、掲載状況は今後も更新します。

役割:

- 厚い 3 ページだけに頼らず、主要ページ全体のコンテンツ密度を上げる

### 4. 通知登録ページ `site/notifications/register.html`

#### 追加ブロックA: 通知の使い方

配置:

- ヒーロー下、登録フォームの前

文案:

通知は、登録した市区町村と、その親の都道府県に関係する選挙だけを受け取るためのものです。
毎日のニュース通知ではなく、地域の選挙を見落としにくくするための補助として使います。
設定後も、通知の停止や登録地域の変更ができます。

#### 追加ブロックB: この通知ページが必要な理由

配置:

- ブロックAの直後

文案:

選挙は必要な時期だけ情報を知りたい人が多く、普段は検索しないまま投票日が近づくことがあります。
このページは、地域登録と端末設定をまとめて行い、必要なときにだけ選挙の存在を思い出せるようにするための入口です。

#### 追加ブロックC: どんな通知が届くか

配置:

- フォームの下か、状態サマリーの上

文案:

通知では、投票日が近い選挙や、確認しておきたい公式案内の更新に合わせて、登録地域に関係する情報だけを届けます。
登録していない地域の通知は届きません。

役割:

- 操作画面だけで終わらせず、サービス内容の説明ページにする

### 5. 固定ページ

必須候補:

- `このサイトについて`
- `情報の集め方と確認基準`
- `プライバシーと通知`
- `お問い合わせ・訂正連絡`

#### このサイトについて

文案:

このサイトは、選挙の公式情報へたどり着きにくい場面を減らすために作っています。
自治体ごとに分散しやすい候補者情報、公報、投票案内を、利用者が知りたい順に並べ直し、まず何を見ればよいか分かる形で届けることを目的にしています。

#### 情報の集め方と確認基準

文案:

掲載情報は、選挙管理委員会、自治体公式サイト、公式 PDF などをもとに確認しています。
内容の正確さには注意していますが、公開タイミングや更新差があるため、最終判断は公式発表をご確認ください。

#### プライバシーと通知

文案:

通知機能では、選択した地域と端末設定に応じて必要な範囲の情報だけを扱います。
個人を特定する目的では利用せず、通知停止や設定変更も利用者自身で行えるようにしています。

#### お問い合わせ・訂正連絡

文案:

掲載内容の更新漏れやリンク切れ、見つけにくい案内があればご連絡ください。
選挙情報は時期によって変わるため、利用者からの指摘も確認の助けになります。

役割:

- 運営実態、確認基準、透明性を明示する

## 既存の厚いページを主導線に乗せる

`site/pages/schedule.html` と `site/pages/turnout.html` は、トップからもっと見えるようにする。

最低限やること:

- トップのナビか本文に `選挙スケジュールを見る`
- `投票率データを見る`

を追加する。

役割:

- 既存の比較的厚いコンテンツを、審査で見えやすい主導線に乗せる

## 実装順

### Phase 1

- トップに 3 ブロック追加
- 固定ページ 2 本以上追加

### Phase 2

- 詳細パネルに定型要約を追加
- 通知登録ページに説明ブロックを追加

### Phase 3

- 個別選挙ページの型を主要選挙へ展開
- トップから schedule / turnout への導線を強化

## 審査前チェック

- 各主要ページに 100-250 字程度の独自本文があるか
- 固定ページが存在し、フッターから辿れるか
- リンク集だけに見えるページが残っていないか
- トップだけでサイトの目的と確認方針が伝わるか
- 通知ページが単なる設定画面に見えていないか
- 個別選挙ページ以外にも十分な読み物ページがあるか
