# わたしの選挙 マスタープラン

最終更新: 2026-04-08

## 1. このプロジェクトの原点

このサイトの出発点は、実際の投票体験にある。

- 市長選に行ったとき、議会の補欠選挙があることを現地で初めて知った
- 選挙情報が自治体サイト、PDF、報道特設ページに分散していて、自分に必要な情報がまとまっていない
- 日本の投票率の低さに対して、「関心の低さ」だけでなく「情報導線の悪さ」が大きな原因だと考えている

このサイトは、政治的に誰かを支持させるためのサイトではない。
有権者が、自分に関係する選挙を知り、迷わず一次情報にたどり着き、投票まで進めるようにするための公共的なナビゲーション基盤を目指す。

## 2. 使命

### ミッション

日本で最もわかりやすく、自分ごととして選挙を把握できるサイトをつくる。

### ビジョン

「自分の地域で次に何の選挙があるか分からない」を日本からなくす。

### 価値提供

ユーザーはこのサイトに来れば、最短で次の3つを把握できる状態を目指す。

1. 自分の地域で次に行われる選挙は何か
2. いつ投票で、いつまでに何を確認すべきか
3. 候補者、公報、投票所、期日前投票などの公式情報はどこか

## 3. 勝ち筋

大手メディアや既存大手と同じ土俵で「速報量」や「政治ニュース量」で勝つのは非現実的。
勝つべき領域は以下の4つ。

### 3.1 自分ごと化

- 郵便番号や住所から、自分に関係する選挙を即座に出す
- 全国ニュースではなく、「あなたの選挙」を最短導線で見せる

### 3.2 地方選挙・補欠選挙の網羅

- 国政選挙だけではなく、知事選、市長選、議会選、補欠選挙まで拾う
- 特に既存サイトで弱い地方議会補選は重要な差別化ポイント

### 3.3 中立性と信頼性

- 全ページに出典、確認日時、更新状態を明示する
- 推測と事実を分ける
- 一次情報への導線を必ず置く

### 3.4 迷わない体験

- 情報量の多さではなく、必要情報への最短到達で勝つ
- 初心者が「結局どこを見ればいいか分からない」とならない設計にする

## 4. 競争戦略

### 4.1 想定競合

- 選挙ドットコム
- NHK選挙特設
- 新聞社の選挙特設
- JAPAN CHOICE系の投票マッチング
- 自治体選挙管理委員会サイト

### 4.2 競合の強み

- 大量の報道・速報
- 候補者DBの蓄積
- メディアパワー
- 選挙期間中の高い更新頻度

### 4.3 競合の弱み

- 地方選挙や補欠選挙の情報が散りやすい
- 常設で「自分の地域の次の選挙」に素早く答える体験が弱い
- 初心者には情報量が多すぎる
- 投票所や期日前投票の導線が自治体PDFに埋もれがち

### 4.4 我々のポジション

「政治ニュースサイト」ではなく、「生活者向け選挙ナビ」

## 5. ターゲットユーザー

### 第一ターゲット

- 20代から40代
- 政治ウォッチャーではない一般有権者
- 投票に行く意思はあるが、情報収集が面倒
- 自分の地域の選挙情報がまとまっていれば使う層

### 第二ターゲット

- 初めて投票する18歳、19歳
- 地方へ引っ越したばかりで土地勘がない人
- 家族や友人に選挙情報を共有したい人

### 使われる状況

- 「今度選挙あるっけ？」と思ったとき
- 投票日直前に投票所や期日前投票を調べたいとき
- 候補者や公報を比較したいとき
- 自治体の選管サイトが見づらいと感じたとき

## 6. プロダクト原則

以下を意思決定の基準にする。

1. 速さより正確さ
2. 意見よりナビゲーション
3. 網羅より信頼できる優先表示
4. デザインより可読性
5. 一次情報への導線を必ず残す
6. 選挙に慣れていない人を基準に設計する

## 7. コア体験

### コアジャーニー

1. ユーザーが郵便番号または住所を入れる
2. 自分の自治体・都道府県・関連選挙区が特定される
3. 次に関係する選挙が時系列で表示される
4. 各選挙ページで候補者、公報、投票日、期日前投票、投票所、選管リンクを確認する
5. 投票直前に必要情報を再確認できる

### 必ず満たすべきUX

- 10秒以内に「自分の次の選挙」が見える
- 選挙ごとに「投票日」「期日前投票」「出典」がすぐ見える
- 候補者比較と選挙公報への導線が迷わない
- スマホで完結する

## 9. データ戦略

### 9.1 基本方針

最も重要なのはUIではなく、全国の選挙情報を正規化するデータ基盤である。

### 9.2 主なデータ種別

- 郵便番号
- 都道府県・市区町村マスタ
- 選挙マスタ
- 候補者データ
- 選挙公報リンク / ファイル
- 期日前投票所
- 投票所
- 過去の投票率
- 過去の選挙結果

### 9.3 優先する情報源

- 郵便番号: 日本郵便の公式データ
- 投票率 / 過去結果: 総務省、統計局、e-Stat
- 選挙日程: 総務省、都道府県・市区町村の選挙管理委員会
- 候補者 / 公報 / 投票所: 各自治体選挙管理委員会

### 9.4 最大の難所

- 候補者一覧や投票所情報が PDF 中心
- 自治体ごとにページ構造がバラバラ
- 更新頻度と公開タイミングが自治体依存

### 9.5 運用方針

- 取得データには必ず `source_url`, `confirmed_at`, `status` を持たせる
- `official`, `derived`, `manual_reviewed` を区別する
- 不確実なデータは表示しないか、未確認として明示する
- まずは「全国で最低限使える」ことを優先し、その後深掘りする

## 11. 機能ロードマップ

### Phase 1: 全国MVP

目的:
「自分の地域の次の選挙が分かる」をまず成立させる

スコープ:

- 郵便番号 / 住所起点の地域特定
- 全国の選挙スケジュールDB
- 地域ページ
- 個別選挙ページの土台
- 投票率データページ
- 出典、更新日時、確認状態の表示

成功条件:

- ユーザーが10秒以内に自分の次の選挙へ到達できる
- 全国主要自治体で最低限の案内が可能

### Phase 2: 実用化

目的:
投票直前にも本当に使える状態にする

スコープ:

- 期日前投票情報
- 投票所情報
- 選挙公報リンク
- 注目選挙の候補者一覧
- 候補者比較表

成功条件:

- 「投票日・場所・候補者・公報」が1サイトで揃う選挙を増やす

### Phase 3: 信頼で勝つ

目的:
初心者が家族や友人に共有したくなるレベルまで使いやすくする

スコープ:

- 候補者ページの標準化
- 過去結果との比較
- 公約テーマ別整理
- FAQ強化
- 共有しやすいカード表示

### Phase 4: 習慣化

目的:
選挙のある時だけ使うサイトから、日常的に思い出されるサービスにする

スコープ:

- 通知機能
- カレンダー登録
- お気に入り地域
- 「次の選挙まであと何日」ウィジェット

### Phase 5: 日本一化

目的:
常設の選挙インフラとして認知される状態を作る

スコープ:

- 全国自治体の深いカバー
- 補欠選挙の高精度網羅
- 候補者DB拡張
- 外部メディア / 教育機関 / NPO との連携余地

## 12. 信頼性ポリシー

### 必須表示

- 情報源
- 最終確認日時
- 確認状態
- 公式ページへのリンク

### 禁止事項

- 出典不明の候補者情報掲載
- 候補者の恣意的なランキング
- 評価基準が不明な比較
- 推測だけで投票所や日程を断定

### 基本姿勢

- このサイトは、投票先を誘導するのではなく、判断材料へのアクセスを助ける

## 13. KPI

### 北極星指標

- 自分の次の選挙に到達できたユーザー率

### 主要KPI

- 郵便番号検索成功率
- 地域ページから個別選挙ページへの遷移率
- 個別選挙ページから公式情報へのクリック率
- 出典表示率
- 更新遅延件数
- 検索流入数
- モバイル再訪率

## 14. 現状草稿の評価

現行の `index.html` ベースの草稿は、以下の点で価値がある。

- 郵便番号起点というコンセプトが明確
- 「次の選挙」への導線が直感的
- 投票率ページとスケジュールページがすでにある
- 静的サイトとして始めやすい

ただし、日本一を狙うには以下が不足している。

- データが全国網羅ではない
- 2026-04-08 時点で選挙データに古い日付が含まれる
- 地域ページ / 選挙ページの粒度が足りない
- SEO向けの静的ページ量が圧倒的に不足
- 出典、更新日時、確認状態が表示されていない
- 候補者、公報、投票所、期日前投票導線が未実装

## 15. いま決めるべきこと

実装前に決めるべき優先事項は次の通り。

1. サイトの最終コンセプトを「生活者向け選挙ナビ」で固定する

3. データモデルを UI より先に設計する
4. 出典・更新日時の表示を必須要件にする
5. 将来的に静的生成しやすい構成へ寄せる
