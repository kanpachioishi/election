# 公式ソース一覧とカバー方針

最終更新: 2026-04-08  
ステータス: Draft v1 / T01 固定用  
関連: [PRD](/home/shimo/election/docs/specs/PRD.md), [データモデル](/home/shimo/election/docs/specs/DATA_MODEL.md), [運用フロー](/home/shimo/election/docs/runbooks/OPERATIONS_FLOW.md), [MVPタスク分解](/home/shimo/election/docs/plans/MVP_TASK_BREAKDOWN.md)

## 1. この文書の目的

この文書は、MVP における「公式ソース」と「公開時のカバー方針」を固定する。

ここで決めること:

- どの組織のどの種別の情報を、公式ソースとして採用するか
- データ領域ごとに、MVP でどこまでカバーを約束するか
- 何を公開してよく、何を公開してはいけないか
- どの条件なら公開を止めるか

この文書で決めないこと:

- 各 `source_url` の個票一覧
- スクレイピング実装や取得手段の詳細
- 候補者比較や公報要約など、MVP 外の編集機能

## 2. 基本原則

1. 公式ソースがない情報は公開しない
2. 公式ソースが複数あり内容が食い違う場合は、解消するまで公開しない
3. `verified` 以外の情報を主要表示に使わない
4. 郵便番号が曖昧な場合は自治体を決め打ちしない
5. 全国完全網羅を約束しない領域は、未カバーである可能性を明示する
6. 候補者、公報、投票所、期日前投票は、MVP では独立データとして正規化せず、確認済みリンクとしてのみ扱う
7. 公式情報への導線を優先し、要約や独自解釈は後段に置く

## 3. 公式ソースの定義

MVP で採用する公式ソース種別は以下とする。

| 種別 | 説明 | 例 | MVP での扱い |
| --- | --- | --- | --- |
| `official` | 当該情報の一次公開主体が出しているページや PDF | 都道府県・市区町村の選挙管理委員会、自治体公式サイト、日本郵便の公式データ | 最優先で採用する |
| `official_aggregated` | 政府・公的機関が集約して公開する公式情報 | 総務省、統計局、e-Stat | 一次ソースが個別に分散する領域の集約元として採用できる |
| `statistical_source` | 統計・結果・投票率などの公的統計ソース | e-Stat、統計局、総務省の統計公開 | 投票率や過去データに使う |
| `manual_summary` | 公式ソースをもとに人手で要約した内部表現 | 内部メモ、内部整形データ | 公開時の根拠そのものにはせず、必ず元の公式ソースに紐づける |

MVP で原則採用しないもの:

- 報道機関の特設ページ
- 選挙ポータルやまとめサイト
- SNS 投稿
- 候補者本人サイトや政党サイトを、選挙運用情報の一次ソースとして使うこと
- 原典が分からない転載ページ

補足:

- 報道機関やまとめサイトは探索補助として見ることはあっても、公開根拠には使わない
- 候補者本人サイトや政党サイトは、将来の候補者情報拡張では補助的に見る余地があるが、MVP の T01 では対象外とする

## 4. ソース優先順位

同じ情報に複数の候補がある場合、優先順位は以下とする。

1. 当該選挙を所管する選挙管理委員会の公式ページまたは公式 PDF
2. 当該自治体・都道府県の公式サイト内の選挙関連ページ
3. 総務省、統計局、e-Stat などの公的集約ページ
4. 内部整形データ

この優先順位に反する公開をする場合は、`note` に理由を残す。

## 5. データ領域ごとの公式ソース一覧とカバー方針

### 5.1 郵便番号と地域対応

| 項目 | 内容 |
| --- | --- |
| データ領域 | `PostalCodeMapping` |
| 主ソース | 日本郵便の郵便番号データ |
| 補助ソース | 自治体コードや地域コードを確認できる公的資料 |
| MVP カバー | 全国必須 |
| 公開条件 | 7 桁郵便番号を基準に、`resolved / ambiguous / unresolved` を判定できる |
| 禁止事項 | 上 3 桁だけで自治体を推測すること、名称類似だけで自治体を決めること |
| 対象外 | 住所自由入力からの推定、非公式住所辞書による補完 |

### 5.2 地域マスタ

| 項目 | 内容 |
| --- | --- |
| データ領域 | `Region` |
| 主ソース | 都道府県・市区町村を識別できる公的コード体系、日本郵便の郵便番号データ、自治体公式情報 |
| 補助ソース | 内部正規化テーブル |
| MVP カバー | 都道府県・市区町村レベルを全国対象 |
| 公開条件 | 地域名、レベル、親子関係、識別子が一意に定義されている |
| 禁止事項 | 旧自治体や表記ゆれを出典なしで統合すること |
| 対象外 | 政令指定都市の区や投票区までの完全正規化 |

### 5.3 選挙マスタ

#### 国政選挙

| 項目 | 内容 |
| --- | --- |
| データ領域 | `Election` |
| 主ソース | 総務省、国政選挙を所管する公的ページ |
| 補助ソース | 関係する選挙管理委員会の公式ページ |
| MVP カバー | upcoming な対象を全国カバー |
| 公開条件 | 選挙名、対象地域、投票日または主要日程、`source_url`、`confirmed_at`、`status=verified` |
| 禁止事項 | 日程未確定のものを確定表示すること |
| 対象外 | 候補者詳細の完全正規化、開票速報 |

#### 都道府県単位の選挙

| 項目 | 内容 |
| --- | --- |
| データ領域 | `Election` |
| 主ソース | 都道府県選挙管理委員会、総務省などの公的情報 |
| 補助ソース | 都道府県公式サイト |
| MVP カバー | upcoming な対象を全国カバー |
| 公開条件 | 国政選挙と同様 |
| 禁止事項 | 根拠ページなしで任期満了日から投票日を推測すること |
| 対象外 | 候補者比較、公約要約、独立した詳細データベース |

#### 市区町村選挙と補欠選挙

| 項目 | 内容 |
| --- | --- |
| データ領域 | `Election` |
| 主ソース | 市区町村・都道府県の選挙管理委員会、自治体公式ページ |
| 補助ソース | 総務省などの公的集約ページがある場合のみ補助利用 |
| MVP カバー | 確認・収集できたものから掲載。全国完全網羅は約束しない |
| 公開条件 | 対象自治体が一意に確定し、基本情報と `source_url`、`confirmed_at`、`status=verified` が揃う |
| 禁止事項 | 郵便番号が曖昧なまま自治体固有の選挙を断定表示すること |
| 対象外 | 未確認自治体の「選挙なし」表示、全国完全網羅の断言 |

### 5.4 確認済み外部リンク

MVP では、候補者、公報、投票所、期日前投票を独立モデルとして持たず、`ElectionResourceLink` として扱う。

| `kind` | 主ソース | MVP カバー | 公開条件 | 対象外 |
| --- | --- | --- | --- | --- |
| `candidate_list` | 選挙管理委員会の公式ページ・公式 PDF | 確認済みのみ | 当該選挙との紐づけが明確で、URL が有効、`status=verified` | 候補者詳細の構造化、候補者本人サイトの採用 |
| `bulletin` | 選挙公報の公式ページ・公式 PDF | 確認済みのみ | 公式公開物であることが明確、URL が有効、`status=verified` | 公報 PDF の自前保管、全文転載 |
| `early_voting` | 期日前投票の公式案内ページ・公式 PDF | 確認済みのみ | 対象選挙、期間、場所案内の公式根拠がある | 会場データの完全正規化 |
| `polling_place` | 投票所案内の公式ページ・公式 PDF | 確認済みのみ | 対象選挙または地域との紐づけが明確 | 投票所詳細の完全正規化 |
| `other` | 上記以外の補助的な公式案内 | 必要時のみ | 何の案内か説明できる | 性質不明のリンク掲載 |

### 5.5 投票率データ

| 項目 | 内容 |
| --- | --- |
| データ領域 | `TurnoutSeriesPoint`, `TurnoutAgeBandPoint` |
| 主ソース | e-Stat、統計局、総務省の公的統計 |
| 補助ソース | なし |
| MVP カバー | 国政選挙の推移と年代別など、主要な公的統計を優先 |
| 公開条件 | 系列名、対象年、数値、`source_url`、`confirmed_at`、`status=verified` が揃う |
| 禁止事項 | 報道記事のグラフを転記して根拠にすること |
| 対象外 | 市区町村レベルの全国網羅投票率データページ |

## 6. MVP カバー方針

MVP 公開時の保証範囲は以下とする。

- 郵便番号検索は全国を対象にする
- `PostalCodeMapping` と `Region` は全国を対象にする
- 国政選挙は upcoming な対象を全国カバーする
- 都道府県単位の選挙は upcoming な対象を全国カバーする
- 市区町村選挙と補欠選挙は、確認・収集できたものから掲載する
- 候補者、公報、投票所、期日前投票は、確認済みリンクだけを個別選挙ページに出す
- 投票率は、まず国政の主要系列を安定公開する

MVP 公開時に保証しないもの:

- 市区町村選挙と補欠選挙の全国完全網羅
- 候補者詳細の標準化
- 候補者比較
- 選挙公報の独立ページ
- 投票所案内の独立ページ
- 期日前投票の独立ページ
- 速報性を前提にした開票情報

## 7. 文書に必ず従う禁止事項

- 公式ソース不在の情報を公開しない
- `source_url` がないデータを公開しない
- `confirmed_at` がないデータを公開しない
- `status=verified` でない詳細情報を主要表示に使わない
- 郵便番号が曖昧なまま自治体固有選挙を表示しない
- 市区町村選挙が未収集なだけなのに「選挙なし」と断定しない
- ニュース記事やまとめサイトを公開根拠に使わない
- 公式 PDF や公報を無断で自前保管・転載しない
- 候補者や政党を恣意的に評価する文言を入れない

## 8. 例外ルール

### 8.1 公式情報が PDF しかない場合

- 公式サイト配下の PDF であれば採用してよい
- 可能なら掲載ページ URL と PDF URL の両方を保持する
- PDF の内容を独自に全文転載せず、リンク導線を優先する

### 8.2 公的集約ソースと個別公式ソースで役割が違う場合

- 現行の運用情報は、原則として当該選挙を所管する選管を優先する
- 統計・過去データは、e-Stat や統計局などの公的集約ソースを優先する
- 両者が食い違う場合は `verified` にしない

### 8.3 郵便番号から自治体が一意に決まらない場合

- 候補地域選択を出すか、未解決として扱う
- 自治体固有の選挙を自動選択しない
- 未解決を「選挙なし」と見せない

### 8.4 未カバー自治体の扱い

- 「現時点で未確認」または「未掲載」の状態を明示する
- 「この地域には選挙がない」と断定しない
- カバー範囲の注記をトップ、地域ページ、または関連導線に出せるようにする

## 9. 公開ブロック条件

次のいずれかに当てはまる場合、そのデータまたはページは公開しない。

- 公式ソースを特定できない
- `source_url`、`source_type`、`confirmed_at`、`status` のいずれかが欠けている
- 公式ソース同士が食い違い、解消できていない
- 郵便番号が曖昧で、自治体を一意に特定できていない
- URL が切れている、または一時ページで安定していない
- `status=needs_update` または `status=unverified` のまま主要表示しようとしている
- 市区町村選挙の未収集を「選挙なし」と見せようとしている
- `ElectionResourceLink` がどの選挙に紐づくか説明できない

## 10. T01 の完了条件

T01 は、次の状態になったら完了とする。

- 公式ソース種別と優先順位が固定されている
- データ領域ごとの主ソース、補助ソース、公開条件、対象外が定義されている
- 全国保証する範囲と、段階展開にする範囲が分かれている
- 禁止事項、例外ルール、公開ブロック条件が文書化されている
- 後続の T02, T03, T06, T07 がこの文書を参照して実装判断できる
