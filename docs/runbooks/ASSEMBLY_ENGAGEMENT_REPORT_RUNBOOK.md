# Assembly Engagement Report Runbook

更新日: 2026-04-27

## 目的

各市議会ごとの関心喚起資料を、公式資料に基づいて安全に作成するための手順を定める。

この資料は、選挙や議会への関心を高める導入資料であり、特定の候補者、政党、会派を支持・批判する選挙運動資料として作らない。

## 保存場所

作業ファイルは目的ごとに分ける。

- テンプレート: `docs/templates/`
- 作成手順: `docs/runbooks/`
- 作成計画: `docs/plans/`
- 収集したURL、PDF、スクリーンショット: `artifacts/private/sources/assembly-reports/{city-slug}/`
- 草稿や生成途中の紙面: `artifacts/private/batches/assembly-reports/{YYYY-MM-DD}/`
- 公開する完成版: レビュー後に `site/` へ移す

リポジトリ直下に新しい資料ファイルを置かない。

## 1. 対象市を決める

最初に次を確認する。

- 既存の市議会ハブページがあるか
- 直近会期の議決結果があるか
- 会派別または議員別賛否が公開されているか
- 前回市議選の公式結果があるか
- 議会だよりや広報紙があるか

賛否図を作るには、賛否数だけでなく、会派別または議員別の根拠が必要。

## 2. 公式資料を集める

最低限、次のURLを集める。

| 種類 | 必須 | 用途 |
|---|---:|---|
| 市議会トップ | 必須 | 公式導線 |
| 会期日程 | 必須 | いつの議会か |
| 議案一覧 | 必須 | 何が提案されたか |
| 議決結果 | 必須 | どう決まったか |
| 会派構成 | 必須 | 議会構成 |
| 前回市議選結果 | 必須 | 選挙との接続 |
| 会議録 | 推奨 | 発言や論点 |
| 録画中継 | 推奨 | 審議内容 |
| 議会だより | 推奨 | 一般向け整理 |

各URLには、次を記録する。

- タイトル
- URL
- 公開日または対象日
- 確認日
- 使用する箇所
- 原文から読み取れる事実

## 3. 1テーマを選ぶ

テーマは、次の条件で選ぶ。

- 公式資料で議決結果を確認できる
- 市民生活との関係を説明しやすい
- 会派構成や賛否とつなげやすい
- 数字や制度を短く説明できる

よいテーマ例:

- 予算案、補正予算案
- 使用料、手数料、給付金、補助金
- 学校、保育、病院、図書館、公共施設
- 防災、交通、上下水道、ごみ処理
- 賛否が大きく分かれた議案

避けるテーマ:

- 公式資料で確認できない話題
- 個人攻撃に見える話題
- 陣営や会派の主張だけで成り立つ話題
- 数字の根拠が不明な話題

## 4. 票差と議決を分けて扱う

資料では、前回選挙の僅差を使って「投票で議会構成が変わる」ことを説明してよい。

ただし、次のように断定しない。

```text
この3票差が今回の議決結果を変えた。
```

使う表現:

```text
議会の人数構成が変わると、予算や条例の賛否にも影響することがあります。
```

```text
当落線付近の数票差が、議会構成を変えることがあります。
```

## 5. 草稿を作る

`docs/templates/ASSEMBLY_ENGAGEMENT_REPORT_TEMPLATE.md` を使ってMarkdown草稿を作る。

草稿の置き場所:

```text
artifacts/private/batches/assembly-reports/{YYYY-MM-DD}/{city-slug}-engagement-report.md
```

この段階では公開しない。

## 6. A4紙面へ流し込む

`docs/templates/assembly-engagement-report-a4.html` を複製して、草稿の内容を流し込む。

確認すること:

- 見出しが強すぎて本文と矛盾していない
- 票差や賛否数が本文中で同じ
- 出典URLが紙面内に残っている
- QRコードを置く場合、公式ページだけを指している
- A4印刷時に文字がはみ出していない

## 7. レビューする

公開前に最低限、次を確認する。

- 数字が公式資料と一致している
- 日付が西暦または和暦で混在しても意味が明確
- 会派構成が確認日時点か議決当時か分かる
- 議決結果が可決、否決、修正可決、同意など正しく書かれている
- 公式資料へのリンクが切れていない
- 選挙結果と議決結果の因果を断定していない
- 中立性の注意書きがある

## 8. 公開する場合

公開する場合は、草稿から直接 `site/` へコピーしない。

1. 完成版のHTMLまたは画像を作る
2. 出典表と確認日を最終更新する
3. 必要なら市議会ハブページからリンクする
4. `node scripts/validate-data-v1.mjs` を実行する
5. 静的HTMLやCSSを追加した場合は `node --check` などで構文確認する
6. ブラウザで表示と印刷プレビューを確認する

## 9. 情報不足時の扱い

賛否数が確認できない場合:

```text
議決結果は確認できますが、会派別・議員別の賛否は公式資料で確認できないため、この資料では賛否図を掲載しません。
```

前回市議選の当落線が確認できない場合:

```text
前回市議選の公式結果ページを確認中です。票差の説明は、確認できるまで掲載しません。
```

会派構成の時点が不明な場合:

```text
会派構成の確認時点が明確でないため、議決当時の人数としては扱いません。
```
