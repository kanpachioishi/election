# わたしの選挙 LINE画面仕様

> Deprecated: 現在の公開通知導線は LINE ではなく PWA / Web Push です。この文書は旧構想の記録として保持しています。

## 1. この文書の目的

この文書は、LINE通知機能に必要な画面を定義する。

ここで決めること:

- どの画面を作るか
- 各画面の役割
- 主要表示項目
- 主な状態
- 画面遷移

ここで詳しく決めないこと:

- 最終ビジュアルデザイン
- CSS の詳細
- 送信文面そのもの

## 2. 画面の結論

LINE通知MVPでは、主画面を次の 4 つに絞る。

1. LIFF地域登録画面
2. LIFF登録完了画面
3. LIFF通知停止画面
4. 配信管理画面

加えて、重要状態を次の 4 つ定義する。

1. 未登録状態
2. 登録済み状態
3. 一時停止状態
4. 送信前プレビュー状態

## 3. 利用者別の画面

### 3.1 LINEユーザー向け

- LIFF地域登録画面
- LIFF登録完了画面
- LIFF通知停止画面

### 3.2 運用者向け

- 配信管理画面

## 4. 画面詳細

### 4.1 LIFF地域登録画面

- 画面ID: `L01`
- 役割:
  ユーザーが受け取りたい地域を登録、確認、変更する主画面。
- 主なユーザー意図:
  自分の市区町村を登録したい。今の登録内容を確認したい。別の地域に変更したい。
- 想定経路:
  - 友だち追加直後の導線
  - リッチメニューの `地域を登録`
  - リッチメニューの `登録を変更`

主要表示項目:

- 画面タイトル
- 現在の登録状態
- 現在の登録都道府県
- 現在の登録市区町村
- LINE ログイン状態
- 都道府県選択UI
- 市区町村選択UI
- 保存ボタン
- キャンセル導線
- サイトを見る導線

主な状態:

- 未登録:
  - `まだ地域が登録されていません`
  - 都道府県、市区町村の選択UIを空で表示
- 登録済み:
  - 現在値を初期表示
  - `登録変更` として見せる
- 保存中:
  - ボタンを非活性
  - 二重送信を防ぐ
- 保存失敗:
  - 再試行導線を表示

バリデーション:

- LIFF ログイン未完了では保存不可
- 都道府県未選択では保存不可
- 市区町村未選択では保存不可
- 都道府県と市区町村の親子関係が不整合なら保存不可

主なCTA:

- `この地域で登録する`
- `変更を保存する`

主な遷移先:

- LIFF登録完了画面
- LIFF通知停止画面
- 本体サイト

### 4.2 LIFF登録完了画面

- 画面ID: `L02`
- 役割:
  登録または変更が完了したことを明確に伝える。
- 主なユーザー意図:
  どの地域で登録されたか確認したい。

主要表示項目:

- 完了メッセージ
- 登録都道府県
- 登録市区町村
- 受け取る通知の説明
- 変更導線
- 閉じる案内

表示メッセージ例の要素:

- `鹿児島県と姶良市の通知を受け取ります`
- `他の市や県の通知は届きません`

主なCTA:

- `登録内容を変更する`
- `サイトを見る`

主な遷移先:

- LIFF地域登録画面
- 本体サイト

### 4.3 LIFF通知停止画面

- 画面ID: `L03`
- 役割:
  ユーザーが配信を一時停止または再開する。
- 主なユーザー意図:
  通知を止めたい。あとでまた再開したい。

主要表示項目:

- 現在の通知状態
- 一時停止の説明
- 再開の説明
- LINE ブロックとの違い
- 実行ボタン

主な状態:

- active:
  - `地域通知を止める` を表示
- paused:
  - `地域通知を再開する` を表示

注意文:

- 一時停止中は地域登録は残る
- 完全停止したい場合は LINE 側でブロックできる

主なCTA:

- `地域通知を止める`
- `地域通知を再開する`
- `登録内容を見る`

主な遷移先:

- LIFF地域登録画面
- 本体サイト

### 4.4 配信管理画面

- 画面ID: `L10`
- 役割:
  運用者が通知候補を確認し、承認し、送信結果を監査する。
- 主なユーザー意図:
  誤配信なく、地域ごとに通知を送る。

主要セクション:

- 通知候補一覧
- 通知候補詳細
- 送信前プレビュー
- 送信履歴
- エラー確認

通知候補一覧の表示項目:

- candidate_id
- election_name
- notification_type
- target_scope
- target_region
- scheduled_for
- status

通知候補詳細の表示項目:

- election_name
- target_region
- notification_type
- vote_date / notice_date
- message preview
- 主要リンク
- recipient_count_preview

送信前プレビューで必須の表示項目:

- 対象地域名
- 対象件数
- 固定済み対象件数
- 県向けか市向けか
- 除外件数
- 送信例
- 対象外例

送信履歴の表示項目:

- campaign_id
- sent_at
- sent_count
- failed_count
- skipped_count
- created_by
- approved_by

主なCTA:

- `承認する`
- `却下する`
- `送信する`
- `履歴を見る`

## 5. 重要状態

### 5.1 未登録状態

- 状態ID: `LS01`
- 適用画面: `L01`
- 表示内容:
  - 登録がまだないこと
  - 登録すると県と市の通知を受け取れること
  - 登録していない地域には送られないこと

### 5.2 登録済み状態

- 状態ID: `LS02`
- 適用画面: `L01`, `L02`
- 表示内容:
  - 現在の登録地域
  - 変更導線
  - 停止導線

### 5.3 一時停止状態

- 状態ID: `LS03`
- 適用画面: `L03`
- 表示内容:
  - 通知停止中であること
  - 登録自体は残っていること
  - 再開導線

### 5.4 送信前プレビュー状態

- 状態ID: `LS10`
- 適用画面: `L10`
- 表示内容:
  - 対象地域
  - 対象件数
  - 対象外の代表例
  - 文面
  - リンク

## 6. 基本遷移

### 6.1 ユーザー導線

1. LINE友だち追加
2. 返信メッセージから LIFF地域登録画面へ
3. 地域登録
4. LIFF登録完了画面
5. 必要に応じて変更、停止、再開

### 6.2 運用者導線

1. 通知候補一覧を見る
2. 候補詳細を開く
3. 送信前プレビューを確認する
4. 承認する
5. 送信する
6. 履歴と結果を見る

## 7. MVP外に回す画面

初期は次を独立画面にしない。

- 登録履歴詳細画面
- 配信監査ダッシュボードの詳細分析画面
- 自由入力住所解釈画面
- 複数地域登録画面
- 通知種別の細かいON/OFF画面

## 8. 画面仕様の結論

LINE通知MVPでは、ユーザー向けは `登録`, `完了`, `停止` の3画面、運用者向けは `配信管理` の1画面で十分である。

重要なのは画面数を増やすことではなく、

- 誤配信しないこと
- 登録地域が明確であること
- 停止と再開が分かりやすいこと
- 送信前に対象地域と対象件数を必ず確認できること

である。
